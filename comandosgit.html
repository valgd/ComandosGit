Comandos de Git

Comando Descripción Parámetros Ejemplo
Configuración
git config --global user.name "nombre" Configuración del nombre de usuario
  • --global: indica que la configuración se aplicará de manera global en tu sistema.
  • user.name: nombre de la clave que estás configurando, en este caso el nombre de usuario
  • "nombre": reemplazar esta parte por tu nombre real
 git config --global user.name "Valeria Gutiérrez"
git config --global user.email "email" Establecer dirección de correo electrónico que se asociará a cada commit y se guarda en el historial
  • --global: indica que la configuración se aplicará de manera global en tu sistema.
  • user.email: nombre de la clave que estás configurando, en este caso la dirección de correo electrónico
  • "email": reemplazar esta parte por dirección de correo electrónico
 git config --global user.email "vadigutierrezde@ittepic.edu.mx"
git config --global -e Fichero que utiliza para editar la configuración global de Git
  • --global: Indica que deseas editar la configuración global de Git.
  • -e: indica que deseas abrir el archivo de configuración en un editor de texto para realizar cambios.
 git config --global -e
Inicialización
git init Inicializar un directorio existente como repositorio Git Ninguno git init
git clone "url" Se utiliza para clonar (copiar) un repositorio Git desde una URL remota a tu máquina local.
  • "url": url del repositorio remoto
 git clone https://github.com/valgd/iw2023ittepic.git
git clone "url" Se utiliza para clonar (copiar) un repositorio Git desde una URL remota a tu máquina local.
  • "url": url del repositorio remoto
 git clone https://github.com/valgd/iw2023ittepic.git
Cambios
git status Muestra el estado de los archivos en tu directorio de trabajo.
  • Ninguno
 git status
git add "fichero" Añadir archivo
  • fichero: nombre del archivo a agregar
  • . : en lugar de agregar un solo fichero, se añaden todos
 git add fichero1.text
git add .
git commit -m "mensaje" Crea un nuevo commit con los archivos en el área de preparación y un mensaje descriptivo.
  • -m: mensaje del commit
 git commit -m "Primer commit"
git diff Muestra las diferencias entre tu directorio de trabajo y la versión más reciente confirmada.
  • ninguno: muestra lo ya mencionado en la descripción
  • nombre de dos commits: muestra la diferencia entre dos commits específicos
 git diff
git diff asdc12 brdf34
git log Muestra todos los commits en el historial
  • ninguno: muestra todos los commits
  • -n "n": muestra los últimos "n" commits
  • --stat: Muestra estadísticas resumidas de cambios en cada commit
  • --oneline: Muestra cada commit en una sola línea con su resumen y el autor.
git log
git log n 5
git log --stat
git log --oneline
git reset Mueve la rama actual y el HEAD a un commit específico
  • --soft: mover el HEAD y la rama actual a un commit anterior y hacer que parezca como si nunca hubieras realizado los commits posteriores
  • --hard: Se utiliza para mover el HEAD y la rama actual a un commit anterior y eliminar todos los cambios realizados en los commits posteriores
git reset --soft HEAD~2
git reset --mixed HEAD~2
git stash Guarda temporalmente cambios no comprometidos en una pila de cambios
  • list: muestra la lista de stashes
  • save: guarda los cambios en un stash con mensaje
git stash list
git stash save "Cambios"
Ramas
git branch Muestra una lista de todas las ramas en el repositorio.
  • ninguno
 git branch
git branch "nombre" Crear una nueva rama en el commit actual
  • "nombre": nombre de la rama a crear
 git branch ramaE
git checkout "nombre" Cambiar de rama
  • "nombre": nombre de la rama a la cual se moverá
 git checkout ramaE
git merge "nombre" Fusiona la rama actual con "nombre".
  • "nombre": nombre de la rama con la que se fusionará la rama actual
 git merge ramaS
git branch -d "nombre" Elimina la rama indicada
  • "nombre": nombre de la rama a eliminar
 git branch -d ramaS
git rebase "nombre" Reorganiza y aplica commits en la rama actual
  • "nombre": nombre de la rama
git rebase main
git cherry-pick Aplica un commit específico de una rama a otra
  • "sha del commit": identificador del commit
git cherry-pick ascd123
Tags
git tag Visualizar todos los tags
  • Ninguno
 git tag
git tag -a "id tag" -m "mensaje" Crear un tag para el último commit realizado con un mensaje enlazado a este tag
  • -a:ndica que estás creando una etiqueta anotada, que incluye un mensaje descriptivo junto con la etiqueta.
  • "id tag": nombre que deseas para tu etiqueta
  • -m "mensaje": mensaje descriptivo que explique la razón o el propósito de la etiqueta.
git tag -a v1.0 -m "primera versión"
git tag -d "id tag" Eliminar el tag especificado
  • -d: eliminar
  • "id tag": nombre del tag a eliminar
 git tag -d v1.0
Compartir y actualizar
git remote add origin "URL_del_repositorio" Conectar el repositorio local a un repositorio remoto
  • origin: : Es el nombre que se le da convencionalmente al repositorio remoto principal. Puedes usar otro nombre si lo deseas
  • "URL_del_repositorio": dirección URL del repositorio remoto que deseas agregar
 git remote add origin https://github.com/valgd/iw2023ittepic.git
git fetch Descarga información sobre las ramas remotas sin fusionar cambios.
  • nombre del repositorio: obtener cambios de un repositorio remoto específico.
  • rama específica: especificar una rama específica en el repositorio remoto desde la cual deseas obtener cambios.
  • todas las ramas:btener todos los cambios y referencias de todas las ramas en el repositorio remoto
  • tags:obtener etiquetas (tags) del repositorio remoto
  • pruning:eliminar ramas locales que ya no existen en el repositorio remoto
  • mostrar progreso:ver un progreso detallado de la descarga
git fetch origin
git fetch origin ramaE
git fecth --all
git fetch --tags
git fetch --prune
git fetch --progress
git push "repositorio"" "rama" Se utiliza para enviar tus cambios locales a un repositorio remoto
  • "repositorio": nombre del repositorio remoto al que deseas enviar tus cambios
  • "rama": ama local que deseas enviar al repositorio remoto
  • --force: forzar la actualización de la rama remota con la versión local
 git push origin ramaE
git push origin ramaE --force
git pull Obtener y fusionar los commits de la remota a la rama local
  • nombre del repositorio: recupera cambios del repositorio remoto llamado
  • nombre de la rama remota:obtiene cambios de la rama remota que está asociada con la rama local actual
  • --rebase:reorganizará tus commits locales sobre los commits remotos
git pull origin
git pull origin ramaE
git pull --rebase origin ramaE

Conclusión personal

En conclusión, GIT es una herramienta muy valiosa para realizar las buenas prácticas de controlar vesiones, este nos proporciona una lista muy grande de comandos a ejecutar que nos facilitan el control al momento del desarrollo.
